# MCS Conversation Viewer — Project Knowledge

> This document is for Copilot/developer onboarding. NOT committed to git.
> Location: project root (gitignored)

## What This Is

A **Power Apps Code App** (React + TypeScript + Vite) that lets makers and admins review Copilot Studio conversation transcripts. It reads from the `conversationtranscript` Dataverse table and presents a debug-friendly view of what happened in each conversation.

## Tech Stack

- **React 19** + **TypeScript 5.9** + **Vite 7**
- **@microsoft/power-apps** SDK for Dataverse data access
- **@microsoft/power-apps-vite** plugin for local dev (Local Play URL)
- **Vitest** for testing (36 tests)
- Plain CSS (no UI framework — we intentionally removed Fluent UI)
- GitHub Actions for release automation, Dependabot, CodeQL

## Dataverse Tables Used

| Table | Purpose | Config |
|---|---|---|
| `conversationtranscript` | The main data — conversation content | power.config.json → `conversationtranscripts` |
| `bot` | Agent display names (resolves schema names) | power.config.json → `agents` (entitySetName: `bots`) |
| `aaduser` | Virtual table — user display name resolution | power.config.json → `microsoftentraids` (entitySetName: `aadusers`) |

## Transcript Content Structure

The `content` field is a JSON string with `{"activities": [...]}`. Key activity types:

| Type | Name/ValueType | What we extract |
|---|---|---|
| trace | ConversationInfo | Session outcome, locale |
| trace | SessionInfo | Duration, turn count, outcome |
| message (role=0) | — | Bot responses (markdown) |
| message (role=1) | — | User messages |
| event | DynamicPlanStepTriggered | Agent thinking/reasoning |
| event | DynamicPlanStepBindUpdate | Tool arguments |
| event | DynamicPlanStepFinished | Tool results + execution time |
| event | DynamicServerToolsList | Available MCP tools |
| event | UniversalSearchToolTraceData | Knowledge sources searched |
| event | ResponseGeneratorSupportData | Generated response, citations |
| trace | KnowledgeTraceData | Search completion state |
| invoke | message/submitAction (feedback) | Reactions 👍/👎 |
| trace | ErrorTraceData | Errors (advanced mode) |
| event | DynamicServerError | MCP server failures (advanced mode) |
| trace | VariableAssignment | Variable changes (advanced mode, filtered: Topic.CurrentTime excluded) |
| trace | DialogRedirect | Topic switches (advanced mode) |
| trace | EscalationRequested / HandOff | Escalation events (advanced mode) |
| event | DynamicPlanStepBlocked | Content filtering / RAI (advanced mode) |
| message | (with attachments) | Adaptive cards, OAuth cards |

## Important Design Decisions

### No $skip — Dataverse doesn't support it
Pagination uses `maxPageSize` + `skipToken`, NOT `skip` (causes error `0x80060888`).

### Bot ID mismatch
The `_bot_conversationtranscriptid_value` lookup field does NOT match `botid` from the bots table. We match on `metadata.BotName` (schema name) instead for agent filtering.

### SharePoint content redaction
Copilot Studio strips bot responses that contain SharePoint knowledge content from the `text` field (security). The response IS available in `ResponseGeneratorSupportData.response`. We show a 🔒 indicator and the knowledge section has the full response.

### Reactions have two formats
- **pva-studio**: `feedback` is an object `{ feedbackText: "..." }`
- **msteams**: `feedback` is a double-encoded JSON string `"{\"feedbackText\":\"...\"}"`

### Orphan reactions
Teams users can react to messages from previous sessions. The `replyToId` points to a message not in the current transcript. We show these in a separate section with on-demand lookup (`contains(content, '<messageId>')` search).

### User identity fallback
If no user messages exist (user opened bot but never typed), the AAD Object ID comes from the `startConversation` event.

### OData injection protection
All user inputs are sanitized via allowlist: `[a-zA-Z0-9\s\-._@:]`. GUIDs are sanitized to `[a-fA-F0-9\-]`.

### Filter state lifted to App.tsx
All list page filter state (date range, search, agent, outcome, feedback, user search query) lives in `App.tsx` as `ListFilterState` so it persists when navigating to detail view and back.

## Dataverse API Notes (Power Apps Code Apps)

- `ConversationtranscriptsService.getAll()` wraps calls in `$batch` requests internally
- Result shape: `{ success: boolean, data: Record[], skipToken?: string, error?: unknown }`
- The SDK uses `Prefer: odata.maxpagesize=N` for page size control
- Local dev requires the **Local Play URL** printed by `npm run dev` — not just localhost
- The `powerApps()` Vite plugin sets `POWERPLATFORMTOOLS_PACPATH` env var in CI
